First we have to sign up for AWS free tier account -

  • Go to https://aws.amazon.com/free
  • Click on Create an account and follow the instruction.
  • Or watch this video for step by step guide on creating account -

Create and launch an EC2 server -

Amazon EC2 provides us a virtual server running on cloud. We can install an operating system on EC2 server to create a virtual machine on cloud, these virtual machines are called EC2 instance. We will use an ec2 instance to deploy and run our Machine Learning model.

After the sign up is complete go to the search bar and type ec2.

Select EC2 from search results.

Click on Launch instance

  • Now we will setup our EC2 instance in 7 steps.

Step 1

Type ubuntu in search bar then select Ubuntu Server 18.04

Step 2

On this page t2.micro (free tier eligible) should be selected by default. Don't change anything on this page. Click on Next.

Step 3

We don't have to change anything on step 3. Click on next.

Step 4

In this step we select how much disk storage our server will have. By default it is set to 8GB which is sufficient for us. So don't change anything on this page. Click next.

Step 5

Click on Add Tag button. Then type “Name” in key section and “webserver-1” in value section (webserver-1 will be the name of your ec2 instance. You can write any other name if you want).

Click Next.

Step 6

Click on Add Rule button. A new row will be created in rules.

In Type select 'All traffic' and in Source select 'Anywhere' . Now click on Review and Launch.

Step 7

On this page we can review our instance details. Don't change anything, click on Launch.

After we click on Launch following page will come.

On this page we will create a private key file, which will be used to verify login from pc to server.

Select Create a new key pair on first option. Then it will ask a name for private key file. Write webserver-passkey in that field and click on Download. Save the downloaded file at a good location as we will use this to login in further steps. Click on Launch Instance.

Now the setup is complete and our instance is launching. On the next page scroll to the bottom left and click on View instance.

Here you can see all EC2 instances in your AWS account.

Pay attention to the Instance state column, it shows us the current state of every instances (virtual servers). If your instance state is Pending that means it is still launching. It might take some time to launch. When the instance state is Running, our instance is launched and ready to connect.

Now we will install some tools to manage EC2 server from our pc -

  • PuTTY - It is a terminal / console which will be used to run commands on our server.

    Download and install putty from this link -

    https://the.earth.li/~sgtatham/putty/latest/w64/putty-64bit-0.76-installer.msi

  • WinSCP - It is a file transfer tool. We will use this to upload files from our pc to EC2 server.

    Download and install winscp -

    https://winscp.net/download/files/2022010418179a0b9943ae09e83e1ae445837d5133d4/WinSCP-5.19.5-Setup.exe

Now we will connect our pc to EC2 server using Winscp and Putty -

Now from instances select (checkbox) webserver-1 and click on Connect.

Now on this page click on SSH client and copy Public DNS.

This public dns is public address of your ec2 instance. We can use this address to access our ec2 instance from anywhere.

Now open winscp in your pc. This scrren will open -

  • In Host Name paste the public dns copied in previous step.
  • In User name we will paste the username of our instance. You can check the user name by going to EC2 Instance Connect. Most probably your username should be “ubuntu”.

  • We don’t have to enter anything in password field. Click on Advanced button below the password box.

    In left panel click on authentication then click on three dots as shown in image -

An explorer window will open. Go to the folder where you had downloaded the private key while setting up instance.

Click on filetypes and select All files.

Now it will show the private key. Select it and click Open.

Our downloaded private key is in .pem extension and winscp supports .ppk extension so it will ask to convert .pem to .ppk, click okay.

Now our .pem file has been converted to .ppk and winscp will ask to save it somewhere. Click on save. (Save this file at a safe place because it will be required by winscp everytime you login.)

After you save the key will be selected and it will show as it is showing in the image below. Now click on OK.

Now on the next screen click on Login button. Winscp will start connecting to the server.

If it asks for any permission allow it by clicking yes.

After winscp finishes connecting it will show following window -

On the left side it shows folders in our pc and on the right side it shows files and folders stored on our ubuntu server. It’s blank because we have not uploaded anything yet.

As you can see in above image I’ve my ML project folder open on right side. You have to open your own project folder.

Now We have to transfer our project files on ubuntu server which is pretty simple. Just drag and drop files from left to right or right click on a file and select upload. It will ask for confirmation, click Ok/Yes.

We have uploaded all required files ubuntu server.

*The required files to deploy our models are -

  • app.py - This is the flask script that connects our ML model to HTML websites and handles input and output of data.
  • index.html - This is the HTML website which will be used to take user input and display result generated by ML model. We have put index.html file in a folder named templates (as you can see in above image).
  • model.pkl - This is the .pkl file of our ML model. It is generated by pickle.

    If you don’t know how to create flask script, html website or pkl file, refer to this video -

    https://youtu.be/_rwNTY5Mn40

  • requirments.txt - This is a simple text file in which we have written all the required libraries needed to run our ML model.

Create a text file, rename it to requirements and type all libraries like this -

My model only requires above mentioned libraries to run. You have to add libraries according to your project’s requirement.

Other than these four files if your project needs some more files to run, you have to upload those on server as well.

Now we will open a console and use it install programs on ubuntu server -

Click on this circled button, it will open Putty console on server.

This is a console running on our ubuntu server (EC2 instance). Keep in mind that it is a linux based console so only linux commands will work on it.

Now we will run some commands to install python and other required libraries.

  • To install Python 3 type this command in console and press enter -
sudo apt install python3
  • Now we will install pip, type this in console and press enter-
sudo apt-get update && sudo apt-get install python3-pip

It will ask if you want to continue, type y and press enter.

  • Now we will use the requirement.txt file to install all libraries with just one command. Type this command and press enter -
pip3 install -r requirements.txt

It will install all libraries and tools that you have written in requirements.txt file.

Now that all required files are installed we are ready to run our ML model on ubuntu server -

Type this command and press enter - 

python3 app.py

It will execute the app.py file that we have uploaded on ubuntu server.

You will see a screen like this. Notice the last line in console, it means our application is running on ubuntu server.

Now go back to AWS and copy the Public DNS address. This address will be used to connect to your app as long as it is running in console.

Paste that address in your browser’s address bar and type “:8080” in the end (as I’ve done in below image). Now press enter.

Our website is successfully loaded now.

This website is connected to our ML model through the flask app. We can enter data on this website to test and demonstrate our ML model.

After you have finished testing your model. Go back to EC2 instances page, select your instance, click in Instance State (as shown in image). Now click on Stop Instance to stop your server.

  • When you want to test your model again just select your instance and start it. Then follow the steps from ‘connecting our pc to ec2 instance using winscp’ till last.
